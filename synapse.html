01 — Definition

What is SYNAPSE?

A methodology where a human director and dozens of AI agents work in continuous coordination. Not an app. Not a plugin. A complete operating model for AI-augmented work.

🧠

Distributed artificial brain

Multiple specialized AI agents working in parallel, each with defined scope, memory, and rules. Together they act as a single coordinated intelligence that never sleeps.

⚡

One sentence. Full execution.

The Service Manager receives a directive. In minutes, a full plan is drafted, validated by a second AI, approved by a human, and dispatched to the appropriate architect. Execution begins.

📡

Across machines. In real time.

The SM runs on Windows. Architects run on a Ubuntu server thousands of kilometers away. Orders flow. Results return. Same channel. Under 60 seconds latency.

🚫

Not a chatbot. Not an assistant.

There is no "chat with AI" button. There are roles, rules, memory, verification gates, and hierarchies — the same structures found in high-performing human teams.

05 — Quality Control

Quality Enforcement

AI agents drift by nature. Without guardrails, they hallucinate, skip steps, and take shortcuts. Boris is the enforcement layer — a system of hooks, gates, and rules that makes quality non-negotiable.

boris-hook — pre-commit gate
$ git commit -m "feat: add payment integration"
# Boris pre-commit hook activates...
→ Checking last-verification.md...
✗ BLOCKED: No evidence found for this change.
Expected: boris_verify() called with concrete result
Found: Estado: PENDIENTE
 
# Agent provides evidence via MCP:
boris_verify(
what_changed: "Payment webhook handler in /api/payments.ts",
how_verified: "curl -X POST http://localhost:3000/api/payments -d '{amount:100}' → 200 OK",
result: "Transaction logged: ID TXN-7842, status: pending_confirmation"
)
 
✓ Estado: APROBADO — Evidence accepted
✓ Commit allowed. Writing to .brain/last-verification.md
$ _
🔒
No commit without evidence
Every git commit requires a prior call to boris_verify with concrete output. The hook blocks if missing.
📋
30-minute memory checkpoints
Agents auto-save state every 30 minutes. If context resets, they resume exactly where they left off.
🎯
Scoped roles — no lane crossing
Architects plan and delegate. Sub-agents execute within a single defined task. Role boundaries are enforced by rules files.
⏱️
Quality gates before dispatch
Every plan requires AI validation (Gemini) + human approval before an architect receives a task. Zero unauthorized executions.

07 — Auditing

The Watchful Auditor

OpenClaw runs 24/7. Every email, search query, form submission, and agent decision is logged. Unauthorized actions are detected and reversed. A daily report arrives every evening.

👁️
Full audit trail
Every action any agent takes is recorded with timestamp, agent identity, and outcome. Nothing is invisible.
⚡
Auto-revert on violation
Agents that act outside their authorized scope trigger an automatic rollback. Rules are enforced, not suggested.
📊
Daily 7PM report
Every day at 7PM, a summary is delivered: tasks completed, exceptions caught, anomalies flagged. The human stays informed without being in the loop constantly.
🔗
Telegram integration
Alerts and status updates arrive via Telegram in real time. The director is never more than a notification away from the current system state.
🧪
Quality checkpoint
Before any work is considered done, OpenClaw validates outputs against the original task definition. No self-reported success without verification.
📡
WebSocket gateway
Loopback WebSocket at 127.0.0.1:18789 connects all agents to the audit bus. Every message flows through a single observable channel.

09 — In practice

One sentence in.
Full plan out.

From a single directive to a fully validated, approved, and dispatched execution plan — the SYNAPSE protocol converts intent into coordinated action.

1

SM creates the plan

The Service Manager receives a one-line directive. SYNAPSE expands it into a full wave-based execution plan with agents, dependencies, and verification criteria.

Claude Opus — Windows
2

Second AI validates

Gemini receives the plan and validates it against 6 mandatory labels: PLAN, TAREA, MODELO, BORIS, VERIFICACION, EVIDENCIA. Gaps are rejected.

Gemini — Cross-validation
3

Human approves

Carlos reviews the plan. No agent moves until explicit approval is given. This is the last human gate before execution begins.

Human checkpoint
4

Architect executes

The SM sends the task via sm-tmux. The architect reads from the Knowledge Hub, spawns sub-agents in parallel waves, and begins execution. Boris enforces quality throughout.
